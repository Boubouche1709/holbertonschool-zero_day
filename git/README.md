# Git Project - Source Code Management

## Project Description

This project is designed to help you understand the fundamentals of source code management using Git and GitHub. By the end of this project, you will have learned how to work with Git for version control, GitHub for hosting repositories, and collaborating on a project.

## Key Concepts

- **Source Code Management (SCM):** This is the practice of managing and tracking changes to a codebase over time. It helps developers to collaborate, keep track of changes, and revert to previous versions if needed.
  
- **Git:** Git is a distributed version control system that allows developers to track and manage changes to a codebase. It helps with versioning, branching, and merging.

- **GitHub:** GitHub is a web-based platform that hosts Git repositories. It provides tools for collaboration, code review, and issue tracking.

## Difference Between Git and GitHub

- **Git** is a version control tool, while **GitHub** is a hosting service for Git repositories. Git can be used locally, whereas GitHub is a cloud-based platform that facilitates collaboration and sharing of code.

## Getting Started

Follow the steps below to set up your project using Git and GitHub:

### 1. Create a New Git Repository
- Use the `git init` command to create a local repository.
- Use `git remote add origin <URL>` to link it to a remote GitHub repository.

### 2. Adding Files and Making Commits
- Use `git add <file>` to add files to the staging area.
- Commit your changes using `git commit -m "Your commit message"`.

### 3. Pushing Code to GitHub
- Push your local commits to the GitHub repository using `git push origin main`.

### 4. Pulling Updates from GitHub
- Use `git pull origin main` to pull the latest changes from the remote repository.

### 5. Creating and Merging Branches
- Create a branch using `git branch <branch-name>`.
- Switch to the new branch using `git checkout <branch-name>`.
- Merge branches using `git merge <branch-name>`.

### 6. Collaborating with Others
- Add collaborators to your repository on GitHub.
- Use pull requests to review and merge contributions from others.

## Files You Shouldn't Commit

- **Sensitive Information:** Never commit files containing passwords, API keys, or other sensitive data.
- **Compiled Files:** Don't commit binaries or compiled code unless explicitly required.
- **Log Files:** Avoid committing log files or temporary files generated by your environment.

## Useful Git Commands

- `git status`: View the current status of your working directory.
- `git log`: View the commit history.
- `git diff`: See changes in your working directory compared to the last commit.
- `git reset`: Undo changes made to files or commits.

## Best Practices

- Write clear and concise commit messages.
- Use branching to work on new features or fixes, then merge when complete.
- Regularly pull updates from the remote repository to stay in sync with the team.
- Keep your repository clean by excluding unnecessary files using `.gitignore`.

## Conclusion

By following these steps and concepts, you will be able to efficiently manage source code using Git and GitHub, work collaboratively, and maintain a clean project history.
